# DataMatcher（画面入力・DB比較検証ツール）

画面（フォーム）に入力された値をJSONとして取り出し、MySQLのSELECT結果（TSV）と突き合わせて **差分/一致** を確認するための、ブラウザで動くローカルツールです。

## 前提

- **実行環境**: ブラウザ（ローカルで `index.html` を開いて利用）
- **入力1**: 画面入力値（JSON）
- **入力2**: DBのSELECT結果（MySQL Workbench等からコピーした **タブ区切りTSV**）

## できること（概要）

- **JSONとDBの値比較**
  - JSON側の `name` からDBカラム名を推定し、値を比較します（OK/NG表示）
- **リスト（`[index]`）の複数行突き合わせ**
  - `list[0].xxx`, `list[1].xxx` のような `name` を **indexごとにグルーピング**し、DBの複数行に割り当てて比較できます
- **突き合わせキー（同一行の判定）を選択**
  - 「この項目が同じなら同じ行」とみなすキー（`name`の末尾）を選べます
- **差分の一覧**
  - JSONにのみ存在する項目、DBにのみ存在するカラムを一覧表示します（DBのみは「何行目か」も表示）

## 使い方

### 1. ブックマークレット（DataCopy）を登録する

1. `index.html` を開く
2. 画面下部の「DataCopy」ボタンを **ブックマークバーにドラッグ&ドロップ**して登録します

### 2. 対象画面でJSONをコピーする（DataCopy）

1. 比較したい入力画面を開く
2. ブックマークバーの **DataCopy** をクリックして実行します
3. クリップボードにJSONがコピーされます

コピーされるJSONの形は概ね次のイメージです。

```json
{
  "formId": {
    "userId": "100",
    "list[0].id": "A01",
    "list[0].name": "Alice",
    "list[1].id": "A02",
    "list[1].name": "Bob"
  }
}
```

### 3. DataMatcherにJSONを貼り付ける

1. DataMatcher（`index.html`）の左側「JSONを貼り付け」にペースト
2. 「比較対象form」で、比較したいフォーム（JSONのトップレベルキー）を選択
3. 「比較対象bean」で、比較したい範囲（bean）を選択
   - **最上位(MainBean)**: フォーム直下の全項目を対象
   - それ以外: `xxx.yyy` 形式の `name` を、最後のドットより前でまとめたグループ（例: `userBean` / `list` など）

### 4. DB結果（TSV）を貼り付ける

1. MySQL Workbench等で、比較対象をSELECTします
2. 結果グリッドからコピーする際は、**カラム名（ヘッダ行）を含めてコピー**してください
3. DataMatcher右側「DB結果（MySQLタブ区切り）」にペーストします

TSVのイメージ（1行目がヘッダ、2行目以降がデータ）:

```text
ID	NAME	USER_ID
A01	Alice	100
A02	Bob	100
```

### 5. （任意）突き合わせキー（name末尾）を選ぶ

リスト形式（`[index]`）でDB複数行に突き合わせる場合、どの値が同じなら同じ行とみなすかを選べます。

- 例: `list[0].id`, `list[1].id` の **`id`** を選ぶ  
  → DB側は自動で `ID` を探し、`ID` が一致する行を割り当てます

ポイント:

- **このプルダウンは「name末尾（leaf）」だけ選びます**
  - DB側カラムは自動で `UPPER_SNAKE`（例: `userId` → `USER_ID`）に変換して探索します
- 未指定でも比較は可能ですが、複数行の割当が「上から順」のフォールバックになります

### 6. 「比較実行」を押す

比較結果が表示されます。

- **比較結果（メイン表）**
  - `OK/NG` で一致判定
  - 左端の「行」列に `JSON[0] ⇔ DB行2` のような割当ラベルが出ます
- **JSONにのみ存在する**
  - JSONにあるが、DBに対応カラムが見つからなかった項目
- **DBにのみ存在する**
  - DBにあるが、JSON側で比較に使われなかったカラム
  - `DB行1 / COL_NAME` のように **何行目の値か**が分かる形で表示されます

### 7. 「NGのみ表示」を使う（任意）

結果表の上部にある「NGのみ表示」をONにすると、NG行だけに絞り込めます。

## 比較ロジックの考え方（概要）

- **対応付け（JSON → DBカラム）**
  - JSONの `name` から末尾（leaf）を取り出し、`UPPER_SNAKE` に変換したものをDBカラム候補にします
  - 例: `list[0].userId` → `USER_ID`
- **複数行（`[index]`）の扱い**
  - `name` に `[0]` のようなindexが含まれる場合、indexごとにJSONをグルーピングし、DBの各行へ割り当てて比較します
- **同一行の判定（突き合わせキー）**
  - 選択された「name末尾」をキーに、DB側の対応カラムで値一致する行を探します（1つのDB行は重複割当しません）

## 改行（CR/CRLF）を含む値について

DB貼り付けTSVの中で、値の中に改行が含まれることがあります。

- **CRLF (`\r\n`)**: 行終端（TSVの行区切り）として扱います
- **CR (`\r`)**: 値の中の改行として扱います
- **CRの直後に付与される半角スペース1つ**は、比較・表示のために削除します
- 比較・表示の段階では、値中の改行は **CRLFとして統一**して扱います

## ファイル構成

- `index.html`: UI本体（入力欄、選択、結果表示、DataCopyリンク）
- `js/index.js`: 比較/表示ロジック
- `css/tailwind.css`: スタイル
